# Grille d'Analyse UX — Audit Visuel Monka

Pour chaque screenshot, vérifier systématiquement ces 8 dimensions.

## 1. Typographie & Wording
- [ ] Pas de fautes d'orthographe
- [ ] Cohérence tutoiement/vouvoiement (pas de mélange dans le même écran)
- [ ] Ton cohérent (pas "votre proche" puis "mon proche" juste en dessous)
- [ ] Texte lisible (taille, contraste, interlignage)
- [ ] Labels clairs et descriptifs

## 2. Hiérarchie visuelle
- [ ] Un élément principal clairement identifiable par écran
- [ ] CTA (Call-to-Action) visuellement proéminents
- [ ] Distinction claire entre contenus de natures différentes (tâches vs conseils vs alertes)
- [ ] Pas de surcharge informationnelle (trop de cartes/blocs simultanés)
- [ ] Espacement suffisant entre les éléments

## 3. Navigation
- [ ] Onglet/menu actif clairement identifié (état sélectionné visible)
- [ ] Tabs fonctionnels (contenu qui change réellement entre onglets)
- [ ] Retour en arrière possible et évident
- [ ] L'utilisateur sait où il est dans l'app

## 4. Personnalisation
- [ ] Utilisation du prénom de l'aidé (pas de "l'aidé" générique)
- [ ] Contenu adapté au profil de l'utilisateur
- [ ] Pas de sections "coquilles vides" (contenu placeholder)

## 5. Pédagogie & Contextualisation
- [ ] Chaque section a une introduction ou un contexte
- [ ] Les termes métier sont expliqués
- [ ] L'utilisateur comprend POURQUOI cette information est là
- [ ] Des micro-explications accompagnent les actions demandées

## 6. Cohérence inter-écrans
- [ ] Style visuel uniforme (couleurs, typographie, espacements)
- [ ] Patterns de navigation identiques entre sections similaires
- [ ] Pas de rupture de design entre deux écrans consécutifs

## 7. États & Feedback
- [ ] Boutons disabled visuellement ET fonctionnellement distincts
- [ ] Messages de succès/erreur présents et clairs
- [ ] États vides gérés (pas de listes vides sans message)
- [ ] Loading states visibles

## 8. Densité & Architecture de l'information
- [ ] Pas de "tout à plat" — des niveaux de profondeur existent
- [ ] Scroll raisonnable (pas de page qui nécessite 15 scrolls)
- [ ] Carousels horizontaux justifiés (pas un cache-misère pour le manque de profondeur)
- [ ] Progressive disclosure : information révélée graduellement
