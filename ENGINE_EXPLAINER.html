🧠 clinicalEngine.ts — 320 lignes

Le cerveau du moteur — scoring + activation

Pure functions, zéro dépendance React. Gère les 9 opérateurs de la DB.

5 fonctions exportées

evaluateCondition()Évalue 1 condition (9 opérateurs : eq, neq, in, nin, gte, contains, count_gte, has_any, ne)
evaluateRule()Évalue 1 règle (logique AND — toutes les conditions doivent passer)
getActivatedCategories()240 règles → catégories activées avec le niveau le plus élevé
computeScore()Score par V (single + multi-choice, cap, seuils)
runEngine()Point d'entrée unique — combine activation + scoring
// Point d'entrée — retourne tout en un seul appel export function runEngine(data, answers, vulnFilter?) { const activated = getActivatedCategories(data, answers) const scores = VULNS.map(v => computeScore(data, answers, v)) const weightedScore = scores.reduce((acc, s) => acc + s.percentage * vulnWeightMap.get(s.vulnId), 0) return { activatedCategories, scores, weightedScore } }
Concepts clés : Compétition des niveaux (standard < ccc < critique), score plafonné par question (QUESTION_SCORE_CAP), score global pondéré (V4 = 30%).

📌 En français

📌 En français

Pattern standard : très courant en TypeScript pour les refactorings — on split le code sans casser les imports.

Health Score /100 — 6 métriques pondérées

Pure TS, zéro React. Mesure la robustesse du moteur, pas la santé du patient.

Les 6 métriques

IDMétriquePoids
M1Couverture questions (dans ≥1 règle)25%
M2Équilibre niveaux (std+ccc+crit par MP)20%
M3Complétude wording (3 niveaux par MT)20%
M4Couverture scoring (questions avec score)15%
M5Complétude acteurs (MT avec acteur[])10%
M6Intégrité FK (liens cat→MP, rule→cat...)10%
// Grade basé sur le score composite function gradeFromScore(score: number) { if (score >= 90) return 'A' if (score >= 75) return 'B' if (score >= 60) return 'C' if (score >= 40) return 'D' return 'F' } // Score composite = Σ(métrique × poids) export function computeEngineHealth(data) { const metrics = [M1, M2, M3, M4, M5, M6] const score = metrics.reduce( (sum, m) => sum + m.value * m.weight, 0) return { score, metrics, grade } }
Transférable directement — c'est du QA embarqué dans le moteur. Outil de monitoring de la qualité des données cliniques.

C3Reco WordingsChaque reco a wording_idec ET wording_utilisateur
